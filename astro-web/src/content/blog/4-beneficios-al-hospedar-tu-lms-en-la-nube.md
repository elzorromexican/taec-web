### 1\. **El poder del SaaS** LMS

Hoy en día el software se vende principalmente como servicio, antes se ofrecía como producto. Eso significa que el cliente obtendrá la última tecnología constantemente  sin tener que adquirir equipo físico. 

No necesitas ocuparte de estar eligiendo la mejor opción para servir a tu aplicación. 

Este servicio implica, Consultoría, capacitación y soporte. Además la posibilidad de contratar servidores elásticos tipo IaaS, la protección a prueba de balas presente en el servicio de hospedaje, Acuerdo de nivel de servicio (SLA), actualizaciones periódicas, garantía de respaldo y ayuda con el diseño gráfico.

Cuando un software se ofrece como servicio (SaaS), todas las cuestiones ya mencionadas pasan a ser asunto del proveedor de software que se haya contratado.

En Taec ofrecemos el software de esta forma. Los servicio que se ofrecen son necesarios, pues de otra manera un proyecto de aprendizaje a distancia no podría tener una funcionalidad tan eficiente. Dicha eficiencia se basa en tres pilares: 

*   Consultoría de implementación de esta forma tú puedes elegir de la mano de profesionales, la opción que más se ajuste a tu realidad.

*   Capacitación directa y continua impartida por la empresa que otorga el servicio para que los docentes e instructores no se rezaguen con la tecnología y puedan sacarle el máximo provecho a esta. 

Soporte 24/7 para los servidores. Estos siempre deberán estar funcionando y si no se puede penalizar a la empresa que presta el servicio por incumplir el acuerdo de nivel de servicio. 

### 2\. Servidores elásticos IaaS para la educación PAYG (Pay as you go)

Al contratar el servicio obtienes lo que se conoce como un servidor elástico. Este sirve para siempre mantener el uso óptimo de servidores, de manera fácil y directa. Paga solo por el uso. Aquí el servidor escala o subescala con pocos clicks para que los gastos dentro del proyecto siempre sean óptimos a través de los cambios en el tiempo.

Este servicio propone también la posibilidad de obtener una valoración automática de AWS.

 Además este servicio permite evitar los gastos fijos, sólo pagar variables, directamente relacionados con el tiempo de uso, Al terminar no hace falta seguir pagando el servicio pues este no se cobra en estado pasivo. 

Al contar con este tipo de servicio es muy facil encontrarse en el umbral ideal de gasto. Además en caso de que la demanda creciera, no se necesitaría instalar nuevos servidores físicos dentro del establecimiento. El caso contrario: no tendrías la posibilidad de estar renovándo pues cada renovación de tecnología implicaría una nueva inversión en hardware.

También, tendría que estar pagando gastos fijos por servicio, evitar esto vuelve la experiencia realmente amigable y justa.

Este servicio hace parte de manejar la infraestructura de hospedaje de la aplicación como un servicio. 

Entre los diferentes elementos que aseguran el nivel de servicio encontramos las métricas siguientes: 

**Utilización de CPU**, supervisión del estado de los volúmenes: el porcentaje de unidades informáticas elásticas como EC2, Kuberneetes y otros asignadas que están actualmente en uso en la instancia. Esta métrica identifica la potencia de procesamiento necesaria para ejecutar una aplicación en una instancia seleccionada.   
_Unidades: Porcentaje_

**Puertas de enlace NAT** (_Network Address Translation_ – Red dirección y traducción), supervisión de Traducción de direcciones de red de entrada de red: la cantidad de bytes recibidos en todas las interfaces de red por la instancia. Esta métrica identifica el volumen de tráfico de red entrante a una sola instancia.   
_Unidades: Bytes_

**Monitor de salida de red y equilibrador de carga clásico**: el número de bytes enviados en todas las interfaces de red por la instancia. Esta métrica identifica el volumen de tráfico de red saliente de una sola instancia.   
_Unidades: Bytes_

**Estado de la memoria caché**: Métricas para monitoreo de memoria caché del procesador.

> La **memoria caché** de un procesador, es un tipo de **memoria** volátil (como la **memoria** RAM), pero muy rápida. Su función es almacenar instrucciones y datos a los que el procesador debe acceder continuamente.
> 
> [https://www.lgblog.cl/lgtv/tecnologia-tendencias/cache/](https://www.lgblog.cl/lgtv/tecnologia-tendencias/cache/)

Informa si la instancia pasó tanto la verificación del estado de la instancia como la verificación del estado del sistema en el último minuto. Esta métrica puede ser 0 (aprobada) o 1 (fallida).   
_Unidades: Cuenta total para aprobar o fallar_.

**Comprobación fallida de estado de instancias**: monitoreo de una instancia de host en la nube:  En computación, cada vez que un programa realiza una acción, se le puede decir que ha realizado una instancia. Este chequeo Informa si la instancia pasó la verificación de estado de la instancia en el último minuto. Esta métrica puede ser 0 (aprobada) o 1 (fallida).   
_Unidades: Cuenta total para aprobar o fallar._

**Comprobación fallida de estado del sistema**: monitoreo del sistema de archivos elástico del host en la nube: informa si la instancia pasó la verificación de estado del sistema en el último minuto. Esta métrica puede ser 0 (aprobada) o 1 (fallida).   
_Unidades: Cuenta total para aprobar o fallar._

Es importante recalcar que la mayoría de métricas de estas instancias, son utilizadas para incrementar el número de las mismas (elasticidad de balanceador de carga), o para el reemplazo en caso de presentar alguna falla. Este reemplazo es en tiempo real, automático y sin afectación al usuario final.

### 3\. **Acuerdo de nivel de servicio (SLA Agreement) de proveedor de SaaS LMS**

![](https://blog.taec.com.mx/wp-content/uploads/2021/05/APROBADO_662x367.jpg)

El acurdo SLA debe incluir: Actualizaciones anuales de versiones, respaldos y diseño gráfico.

Estos índices hacen parte de nuestro SLA (Service level agreement), nuestro acuerdo de nivel de servicio. 

Este acuerdo constituye un punto de referencia para que ambos lados implicados entiendan cuál va a ser el proceso de mejora continua que se va a implementar periódicamente. 

También establece los parámetros de servicio y atención que tú tienes derecho a demandar a la empresa que presta el servicio. Este acuerdo de nivel de servicio se mide a través de 5 parámetros distintos.

*   Ratio de abandono de llamada: Porcentaje de llamadas que no lograron recibir atención al cliente. 
*   Tiempo medio previo a la atención del operador para que atienda la llamada:
*   Porcentaje de atención proporcionada en un tiempo de referencia limitado: este índice denota cuántas llamadas están siendo atendidas dentro de un lapso de tiempo ideal. Por ejemplo: 60 % de las llamadas están siendo atendidas dentro del plazo ideal de 20 segundos.
*   Índice de éxito en atención telefónica a primer contacto: Grado de éxito al solucionar las llamadas en un primer contacto.
*   Tiempo transcurrido para completar una tarea específica referente a la atención al cliente en caso de que se solucione,  cuanto tiempo pasa para que el operador solucione la problemática específica del cliente. 

### **4 .Servicio de capacitación**

Aplica la capacitación de tercero por parte del proveedor para ofrecer un nivel de uso adecuado a los usuarios y poder empezar a avanzar desde ahora. Los usuarios capacitados además necesitan obtener su certificación de capacitación en el uso de la plataforma. De acuerdo con los perfiles a capacitar, sugerimos el siguiente plan de formación por perfil:  

1\. Alumno (todos los usuarios)

*   Guía de acceso a la plataforma
*   Página de preguntas frecuentes

2\. Profesor que no edita (aproximadamente 50 alumnos)

*   Infografía general referente al  recurso y actividad en plataforma
*   Guías de calificación de actividades 
*   Guía para la moderación de foros 

3\. Docente (20 usuarios)

*   Todos o alguno de los recursos de los perfiles anteriores
*   Programa virtual para la creación de cursos básico
*   Programa virtual para la creación de cursos avanzado

4\. Administrador (5 usuarios)

*   Todos o alguno de los recursos de los perfiles anteriores
*   Programa virtual para administrar usuarios y audiencias
*   Programa virtual para la gestión de roles
*   Programa virtual para la gestión cursos y programas
*   Programa virtual para la gestión de informes 

5\. Director (1 usuario)

*   Todos o alguno de los recursos de los perfiles anteriores
*   Programa virtual para administrar usuarios y audiencias
*   Programa virtual para la gestión de roles
*   Programa virtual para la gestión cursos y programas
*   Programa virtual para la gestión de informes 

De esta forma aseguras un entendimiento general del uso de la plataforma con el cual ya es posible proceder y evolucionar de manera escalada.

### **Sistemas de plataforma SaaS LMS a prueba de balas**

Todas las nubes privadas más grandes poseen la posibilidad de hacer implementaciones de manera “sencilla”. Hay que recalcar que esto depende de la complejidad del proyecto, por lo que será esencial contar con profesionales que sepan implementar correctamente la opción. Por sí mismo cualquier nube puede resultar segura o insegura. Aún así existen diferencias entre los distintos servicios que ofrece el mercado. Los más Grandes AWS, Microsoft Azure Y Google Cloud.

Si planeas hospedar tus clases en la nube, asegúrate de estar en un lugar correcto pues existen implementaciones más seguras que otras. 

Por ejemplo, la posibilidad de tener el servidor en Amazon Web Services AWS con los ingenieros de Taec desarrollando su proyecto significa una garantía de servicio a prueba de fallas en el 99.9% de casos en los servidores y a prueba de fallas en el 99.5% de casos en los LMS.

Sin embargo, cuando los ingenieros no son lo suficientemente cuidadosos, pueden causar problemas. Es por eso que el profesionalismo del equipo de implementación del proyecto es un elemento base para el éxito. 

A continuación las tres nubes privadas más conocidas y de mayor reputación actual:

*   Almacenamiento

*   Redes y entrega de contenido
*   Herramientas de desarrollo
*   Herramientas administrativas
*   Seguridad, identidad y cumplimiento
*   Analítica
*   Inteligencia artificial
*   Servicios móviles
*   Servicios de aplicaciones
*   Mensajería
*   Productividad empresarial
*   Transmisión de escritorio y aplicaciones
*   Software
*   Internet de las Cosas

Además de esta amplia gama de opciones, en AWS tienen productos específicos con un alto grado de categorización.

Estas soluciones cubren:

*   Sitios web

Amazon resulta una plataforma la plataforma más intuitiva para construir y diseñar proyectos. 

**Microsoft Azure**

Microsoft posee aproximadamente la misma tecnología que AWS. Aunque poseen la característica de facturación flexible para pagos más personalizados del usuario.

Microsoft propone guías que exploran a profundidad las funciones para que cada quien pueda desarrollar su proyecto. 

**Google Cloud**

Google Cloud posee tres características principales que lo destacan.

*   Infraestructura ultra innovadora para el uso de usuario..
*   Análisis, estadísticas y datos serios. 
*   Puro código, sin servidor.

Cualquiera de los tres servidores debe de contar con expertos en la plataforma ya que funcionan de manera distinta, por lo que la ingeniería de sistemas cambia de una plataforma a otra. Identifica bien tus necesidad y podrá distinguir entre la que más a su proyecto, aunque en realidad usted eligiera a terceros que trabajan con una u otra plataforma.

### **Precios en AWS.**

La novedosa tendencia del PAYG hace que los precios se ajusten a las necesidades del proyecto y todo resulte más barato. No es necesario comprar costosas soluciones de nube privada que se vuelven obsoleta en poco tiempo. Con Amazon Web Services (AWS) solo necesitas ajustar tu máquina virtual y la nube de AWS se adaptará según los requerimientos, para solo pagar por el poder de cómputo usado.

Desafortunadamente cuando se trata de almacenar grande volúmenes de datos (entre 50 TB y 500 TB) AWS pierde rendimiento porcentual.

> Amazon puede ser ideal para colocar grandes bases de datos en la nube pero cuando se trata de subir aplicaciones la nube de Microsoft Azure está mejor equipada.
> 
> Estos detalles de precios de AWS son útiles porque, como Azure y Google, los niveles de niveles de nube varían mucho y es bueno compararlos con los requisitos que necesite nuestro proyecto.
